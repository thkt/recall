## Bug follow-ups (route to /issue, NOT ADRs)

Documenting these as intentional invariants would lock in the bug.

1. **chunker.rs:116-124 silent embed-drop on multi-assistant turns** (verified by critic): the parser drops empty tool_result-only user turns (claude.rs:67), leaving consecutive assistant messages; chunk_messages pairs only the first, orphaning the trailing assistant. Its text reaches FTS but is never embedded → findable by keyword, invisible to semantic search on every multi-assistant tool-use turn. H-impact, kin to the stale-row-agent-invisible class.
2. **main.rs:~452 embed_capable load-vs-inference gap** (#215-adjacent, untracked): embed_capable = load_result.is_ok() gates preserve-vs-rebuild on model _load_ only; a model that loads but fails at inference (corrupt weights) reports capable, takes the rebuild path, and destroys embeddings the #215 fix protects.
3. **search.rs:393-412 MATCH "" per-session warning** on the #167 degraded edge: redundant inner-MATCH on rowid-tier miss + a logged warning per session for short-only queries.
4. **ansi.rs:25-60 tab/CR stripping**: strip_control_chars drops \t and \r (keeps only \n) for a tool emitting transcript/markdown — confirm intentional and document the keep-set, or fix.
5. **date.rs UTC-only rendering**: format_timestamp has no local-tz offset; a JST user sees the prior calendar day for late-night sessions. Confirm intentional (agent-stable output) and document, or fix.
6. **codex.rs:88-90 cwd only in legacy arm**: a modern Codex file with env-context arriving as response_item yields empty `project`. Verify against real corpus.
