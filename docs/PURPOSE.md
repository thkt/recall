# Purpose

## Problem
<!-- 現在の痛み。このプロジェクトがなければ何が困るか -->

過去のClaude Code / Codexセッションから「あのとき何をどう解決したか」を探すのが困難。数千のJSONLファイルをgrepするのは非現実的で、トピックベースの検索ができない。

## Outcome
<!-- 達成後の状態。Problem の裏返しになっても問題ない -->

`recall "query"` の一発で、過去セッションからトピックベースで即座に見つかる状態。プロジェクト・期間・ソースでのフィルタリングも可能。

## Users
<!-- 誰のためのものか。内部ツールなら利用する開発者の役割・状況 -->

- Claude Code / Codexを日常的に使う開発者
- Claude CodeのAIエージェント（ツールとしてrecallを呼び出す）

## Scope

### In

- ローカルのJSONLセッションファイルの全文検索
- FTS5インデックスの増分更新
- プロジェクト・期間・ソースによるフィルタリング
- BM25 + 新しさブーストによるランキング

### Out

- クラウド同期・リモートセッションの検索
- セッション内容の全文表示・エクスポート
- 画像・ツール結果・バイナリコンテンツのインデックス
- GUI

### Accepted Tradeoffs
<!-- 意図的に受け入れている制約・省略 -->
- テキストのみインデックス。ツール使用ブロック・システムプロンプトは除外
- SQLiteを静的リンク（バイナリサイズ > 依存ゼロを優先）
- セッション全文は表示しない（抜粋のみ。全文はJSONLを直接開く）

## Priorities
<!-- コードレベルで検証可能なトレードオフ方針。順序が意味を持つ
     現在のフェーズ: 安定期（v0.2.0 リリース済み） -->

1. 検索速度と精度 > 機能の豊富さ
2. ゼロ依存・シングルバイナリ > 開発の利便性
3. 増分インデックスの正確性 > インデックス速度
